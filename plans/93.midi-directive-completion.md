## Outline

### Goal
Add auto-completion for MIDI directives using the `%%MIDI` prefix format, with context-aware command suggestions and parameter snippets.

### Definition: MIDI Directive

MIDI directives control MIDI playback settings. They use a two-part format: `%%MIDI command [parameters]`. Example: `%%MIDI program 0` or `%%MIDI transpose -2`. The completions must detect when the user is typing after `%%MIDI ` and offer command-specific completions.

### Scope

MIDI commands (42 total) organized by parameter pattern:

No parameters (15):
- `nobarlines`, `barlines`, `beataccents`, `nobeataccents`, `droneon`, `droneoff`
- `drumon`, `drumoff`, `fermatafixed`, `fermataproportional`, `gchordon`, `gchordoff`
- `controlcombo`, `temperamentnormal`, `noportamento`

One identifier (3):
- `gchord`, `ptstress`, `beatstring`

One integer (18):
- `bassvol`, `chordvol`, `c`, `channel`, `beatmod`, `deltaloudness`, `drumbars`
- `gracedivider`, `makechordchannels`, `randomchordattack`, `chordattack`
- `stressmodel`, `transpose`, `rtranspose`, `vol`, `volinc`, `gchordbars`

Two integers (6):
- `ratio`, `snt`, `bendvelocity`, `pitchbend`, `control`, `temperamentlinear`

Program (1-2 integers):
- `program`

Four integers (1):
- `beat`

Five integers (1):
- `drone`

Identifier + integer (1):
- `portamento`

Fraction (3):
- `expand`, `grace`, `trim`

Identifier + multiple integers (2):
- `drum`, `chordname`

Integer + optional octave= (2):
- `bassprog`, `chordprog`

Identifier + drum sound (1):
- `drummap`

All defined in `parse/types/directive-specs.ts` lines 277-379 in `MIDI_COMMAND_SPECS`.

### Context Detection

The completion system must detect when the cursor is:
1. After `%%MIDI ` - offer MIDI command completions
2. After `%%MIDI partial` - filter commands by prefix

This requires extending `getDirectiveCompletionContext` to recognize the MIDI context.

Note on whitespace handling: To be consistent with other directive types, the prefix is trimmed of leading spaces. Typing `%%MIDI   prog` with multiple spaces will result in a prefix of `prog`, not `  prog`.

### MIDI Directive Discovery

Because users need to discover the MIDI directive itself (before typing command names), we add "MIDI" to the general directive completions. When users type `%%MID`, they will see "MIDI" in the completion list. After selecting it and typing a space, they enter MIDI command context and see command completions.

### Prerequisites

This plan depends on plans 89, 90, 91, and 92 being implemented first, because the server.ts handler references functions from those plans.

### Snippet Formats

Commands grouped by snippet pattern:

No parameters:
```
%%MIDI nobarlines
```

One integer:
```
%%MIDI channel ${1:1}
```

Two integers:
```
%%MIDI ratio ${1:3} ${2:2}
```

Fraction:
```
%%MIDI grace ${1:1/8}
```

### Design Decisions

1. Extend `DirectiveCompletionContext` with `midi-command` type
2. Use `InsertTextFormat.Snippet` with appropriate placeholders per command
3. Group similar commands to share snippet patterns
4. Plain text fallback inserts command name only
5. Use `CompletionItemKind.Keyword` for MIDI commands
6. Documentation includes command description and expected parameters

### 1.2 Update getDirectiveCompletionContext

File: `abc-lsp-server/src/completions.ts`

Update the function to detect MIDI context:

```typescript
export function getDirectiveCompletionContext(
  lineText: string,
  charPosition: number
): DirectiveCompletionContext {
  const textBeforeCursor = lineText.slice(0, charPosition);

  // Check if we're after %% at line start
  if (!textBeforeCursor.match(/^%%/)) {
    return { type: "none" };
  }

  // Extract what's after %%
  const afterPercent = textBeforeCursor.slice(2);

  // Check for MIDI command context - must be after "%%MIDI " (case-insensitive)
  const midiMatch = afterPercent.match(/^midi\s+(.*)$/i);
  if (midiMatch) {
    return { type: "midi-command", prefix: midiMatch[1].trim() };
  }

  // Check for specific directive contexts
  const abclsParseMatch = afterPercent.match(/^abcls-parse\s+(.*)$/i);
  if (abclsParseMatch) {
    return { type: "abcls-parse-options", prefix: abclsParseMatch[1] };
  }

  const abclsFmtMatch = afterPercent.match(/^abcls-fmt\s+(.*)$/i);
  if (abclsFmtMatch) {
    return { type: "abcls-fmt-options", prefix: abclsFmtMatch[1] };
  }

  const abclsVoicesMatch = afterPercent.match(/^abcls-voices\s+(.*)$/i);
  if (abclsVoicesMatch) {
    return { type: "abcls-voices-options", prefix: abclsVoicesMatch[1] };
  }

  // If just %% or %%<partial>, offer directive names
  return { type: "directive-name", prefix: afterPercent };
}
```

### 2.2 Add MIDI Completion Function

File: `abc-lsp-server/src/completions.ts`

Add after the MIDI constants:

```typescript
export function getMidiCommandCompletions(prefix: string, supportsSnippets: boolean = true): CompletionItem[] {
  const trimmedPrefix = prefix.trim();
  return MIDI_COMMANDS
    .filter((cmd) => cmd.label.toLowerCase().startsWith(trimmedPrefix.toLowerCase()))
    .map((command) => ({
      label: command.label,
      kind: CompletionItemKind.Keyword,
      documentation: command.documentation,
      insertText: supportsSnippets ? command.snippet : command.label,
      insertTextFormat: supportsSnippets ? InsertTextFormat.Snippet : InsertTextFormat.PlainText,
      data: { type: "midi-command", command: command.label },
    }));
}
```
